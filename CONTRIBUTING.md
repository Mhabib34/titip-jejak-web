Terima kasih sudah tertarik untuk berkontribusi ke TemuKan! 🧡

## 🇬🇧 English

### Code of Conduct

This project is built for humanitarian purposes. We expect all contributors to:

- Be respectful and constructive in all interactions
- Not use this platform or codebase for commercial gain
- Protect the privacy of missing persons data at all times

### How to Contribute

#### 1. Reporting Bugs

Open an issue on GitHub with:

- A clear title describing the bug
- Steps to reproduce
- Expected vs actual behavior
- Screenshots if applicable

#### 2. Suggesting Features

Open an issue with the `enhancement` label. Describe:

- What problem this solves
- How you envision it working
- Any alternatives you considered

#### 3. Submitting a Pull Request

```bash
# 1. Fork the repository
# 2. Create a feature branch
git checkout -b feat/your-feature-name

# 3. Make your changes
# 4. Commit with a clear message
git commit -m "feat: add map clustering for report pins"

# 5. Push and open a PR
git push origin feat/your-feature-name
```

**Branch naming convention:**

- `feat/` — new feature
- `fix/` — bug fix
- `docs/` — documentation only
- `refactor/` — code refactor, no feature change
- `chore/` — config, deps, tooling

**Commit message format** (loosely following [Conventional Commits](https://www.conventionalcommits.org)):

```
feat: short description
fix: short description
docs: short description
```

#### 4. PR Guidelines

- Keep PRs focused — one feature or fix per PR
- Add a description of what changed and why
- Make sure the project builds without errors before submitting
- Be responsive to review feedback

### Sensitive Data Warning

⚠️ Never commit real personal data, API keys, or Supabase credentials.  
Always use `.env.example` as a template and keep secrets in `.env` (gitignored).

Open a GitHub Discussion or email [mhabib34official@gmail.com](mailto:mhabib34official@gmail.com).

## 🇮🇩 Indonesia

### Kode Etik

Proyek ini dibangun untuk tujuan kemanusiaan. Semua kontributor diharapkan:

- Bersikap sopan dan konstruktif dalam semua interaksi
- Tidak menggunakan platform atau kode ini untuk keuntungan komersial
- Selalu menjaga privasi data orang hilang

### Cara Berkontribusi

#### 1. Laporan Bug

Buat issue di GitHub dengan:

- Judul yang jelas menggambarkan bug
- Langkah-langkah untuk mereproduksi
- Perilaku yang diharapkan vs yang terjadi
- Screenshot jika ada

#### 2. Usulan Fitur

Buat issue dengan label `enhancement` dan jelaskan masalah yang ingin dipecahkan.

#### 3. Membuat Pull Request

```bash
# 1. Fork repositori
# 2. Buat branch fitur
git checkout -b feat/nama-fitur-anda

# 3. Buat perubahan
# 4. Commit dengan pesan yang jelas
git commit -m "feat: tambah clustering peta untuk pin laporan"

# 5. Push dan buka PR
git push origin feat/nama-fitur-anda
```

**Konvensi nama branch:**

- `feat/` — fitur baru
- `fix/` — perbaikan bug
- `docs/` — dokumentasi saja
- `refactor/` — refactor kode
- `chore/` — konfigurasi, deps

#### 4. Peringatan Data Sensitif

⚠️ Jangan pernah commit data pribadi nyata, API key, atau kredensial Supabase.  
Gunakan `.env.example` sebagai template, simpan rahasia di `.env` (sudah di-gitignore).
